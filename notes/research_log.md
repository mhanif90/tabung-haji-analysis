\## 26 June 2026 — Day 1



\### Initial Project Setup



Today I officially started my first real-world data analytics project.



Instead of jumping straight into Python or AI, I first learned the importance of version control using Git and GitHub. I created my first Git repository, initialized it using `git init`, and learned that Git acts like a timeline for my project. Rather than saving every tiny change automatically, Git lets me decide when a meaningful milestone should be recorded.



\### Things I Learned



\* The difference between Git and GitHub.

\* `git init` starts Git tracking for a project.\[start tracking files]

\* `git add .` stages selected changes for the next checkpoint.\[hey im gonna include this in my project]

\* `git commit -m "message"` records a milestone into the project's history. \[adding a milestone]

\* `git log` displays the complete history of the project. \[list project history/milestone]

\* Every commit should describe a meaningful milestone rather than simply saying "Update" or "Changes".



\### Reflection



Today I understood that Git is much more than a backup tool. It records the story of a project's evolution. Every commit becomes a chapter in that story.

\## 27 June – 5 July 2026



\### Understanding Git and GitHub



Over the next few days, I spent time strengthening my understanding of Git instead of rushing into coding.



\### Things I Learned



\* Git stores project history locally on my computer. \[personal workspace]

\* GitHub stores a published copy of that history online. \[public library]

\* Local commits do not automatically appear on GitHub. \[Not everything is automatically updated. I choose what's valuable]

\* `git push` uploads local commits to GitHub. \[Update everything in git to GitHub]



\### Git Concepts That Became Clear



\* Working Directory \[pwd]

\* Staging Area \[add .]

\* Commit History \[commit -m 'title']

\* Repository

\* Branch (`master`)

\* HEAD

\* `origin/master`



I also learned that:



\* HEAD represents my current position in the project's history.

\* The branch represents the timeline itself.

\* Git allows me to travel backwards through project history if necessary.

Git only tracks files.



That explains why newly created folders did not appear in `git status` until they contained actual files.
